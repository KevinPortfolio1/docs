負載均衡 (Load Balancing)

在大規模系統中，單一伺服器無法承受所有流量。LB 位於客戶端與伺服器之間：

  • L4 負載均衡： 基於 IP 和 Port 進行轉發。
  • L7 負載均衡： 基於 HTTP 內容、Cookie 進行智慧路由。

緩存策略 (Caching)

如何保持數據一致性是關鍵：

  • Cache Aside： 最常用，應用程式負責維護。
  • Write-Through： 寫入資料庫時同步寫入緩存。

消息隊列 (Message Queues)

用於解決高併發寫入時的衝擊，實現「削峰填谷」：

  • Pub/Sub： 訂閱發布模式，一對多。
  • FIFO Queue： 保證處理順序。
